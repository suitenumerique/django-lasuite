Django La Suite Demo

This demo application showcases how to use django-lasuite to build an OIDC client that authenticates users and then consumes a protected resource server API.

How it works

  1. User clicks Log in with OIDC and is redirected to the OIDC provider.
  2. After successful authentication, the provider redirects back with an authorization code.
  3. The demo app exchanges the code for tokens (access, ID, and refresh tokens).
  4. Tokens are stored in the session so they can be reused.
  5. When calling the resource server, the access token is sent in the Authorization: Bearer <token> header.

Configuration

All OIDC settings are loaded from environment variables. See demo/settings.py for the full list of configurable options.

Environment Variable Purpose
OIDC_OP_URL OIDC provider base URL
OIDC_OP_AUTHORIZATION_ENDPOINT Authorization endpoint
OIDC_OP_TOKEN_ENDPOINT Token endpoint
OIDC_OP_USER_ENDPOINT UserInfo endpoint
OIDC_RP_CLIENT_ID Client ID
OIDC_RP_CLIENT_SECRET Client secret
RESOURCE_SERVER_BASE_URL Base URL of the resource server to consume
